# 售后官网系统 (After-sales Official Website)

基于 **React 19 + Vite + React Router + TailwindCSS + Ant Design** 开发的高性能、响应式售后服务平台。本项目不仅实现了完整的业务流，还在架构设计、安全加固及国际化维护上遵循了企业级开发标准。

## 🚀 技术栈

- **核心框架**: React 19 (最新特性支持)
- **构建工具**: Vite 5 (极速编译体验)
- **路由管理**: React Router 7 (支持嵌套路由、路由守卫及组件懒加载)
- **UI & 样式**: Ant Design 5 + TailwindCSS 3 (原子化样式与组件库完美结合)
- **状态管理**: Zustand + Immer (支持跨标签页持久化存储)
- **国际化**: i18next (JSON 语言包分离，支持可视化表格维护)
- **网络通信**: Axios (集成请求/响应拦截器，支持 401 自动登出)

## 🛠 核心功能模块

1.  **首页**: 品牌展示、动态轮播图、功能快捷入口、最新公告、响应式四栏页脚。
2.  **身份认证**: 基于 Modal 的登录注册体系，支持找回密码、隐私政策跳转及跨 TAB 状态同步。
3.  **个人中心**: 模块化拆分设计，包含服务记录（维修/咨询/投诉）、脱敏后的个人信息及账户安全设置。
4.  **售后维修**: 在线预约流程，支持图片上传、日期选择及进度反馈。
5.  **服务咨询**: 嵌套路由结构，支持电车与储能业务的独立咨询，集成富文本编辑器。
6.  **官方商城**: 商品列表流、详情页展示、完整的购物车管理逻辑。
7.  **加盟与投诉**: 包含多步骤申请表单、资质上传及实时的处理进度查询。

## 🛡 安全与合规

- **数据脱敏**: 自动对手机号、邮箱等敏感信息进行掩码处理。调用 `src/utils/mask.ts` 中的工具函数。
- **传输安全**: Axios 拦截器自动注入 Token，统一处理 401 非法访问并自动清除登录态。
- **Web 防御**: 配置严格的内容安全策略 (CSP)，防止 XSS 攻击；所有表单输入均需进行前端校验。
- **隐私保护**: 完整的隐私政策与服务协议页面，协议链接必须支持新窗口打开 (`target="_blank"`)。

## 📂 项目结构

```text
src/
├── api/          # Axios 请求封装与安全拦截逻辑
├── assets/       # 静态资源（图片、字体等）
├── components/   # 全局级公共组件
├── i18n/         # 国际化配置，locales 目录下包含 JSON 翻译文件
├── layouts/      # 布局组件，定义全站统一视觉框架
├── pages/        # 业务模块
│   └── [Module]/components/  # 模块级私有子组件
├── routes/       # 路由中心，包含权限控制 (ProtectedRoute)
├── store/        # Zustand 状态中心，支持持久化配置
└── utils/        # 工具函数 (数据脱敏、样式合并、通用的逻辑处理)
```

## 💡 开发规范

### 1. 命名与代码风格
- **组件命名**: 统一使用大驼峰（PascalCase），如 `AuthModal.tsx`。
- **文件名**: 与组件名保持一致。子组件存放于所属页面的 `components` 目录下。
- **变量命名**: 使用小驼峰（camelCase），常量使用全大写加下划线（CONSTANT_NAME）。
- **函数式编程**: 优先使用 **React 19 函数式组件** 和 Hooks，禁止使用类组件（除非是 ErrorBoundary 这种特殊场景）。

### 2. 状态管理 (Zustand)
- **单一职责**: 按功能模块拆分 Store（如 `useAuthStore`, `useMallStore`）。
- **不可变更新**: 必须配合 `immer` 中间件进行状态更新。
- **持久化**: 涉及用户登录态、购物车等关键信息必须使用 `persist` 中间件，并合理配置 `partialize` 过滤非必要字段。

### 3. 样式与 UI (TailwindCSS & Ant Design)
- **原子化优先**: 优先使用 TailwindCSS 处理布局、间距和颜色。
- **组件库一致性**: 涉及表单、弹窗、消息提示等复杂交互，必须使用 Ant Design 组件以保证全站体验一致。
- **AntD 样式覆盖**: 使用 Tailwind 的 `@apply` 或 AntD 5 的 `styles` 属性进行定制，避免直接在全局 CSS 中写大段覆盖样式。

### 4. 路由与性能
- **懒加载**: 所有一级页面必须使用 `lazy` 和 `Suspense` 进行动态加载。
- **嵌套路由**: 模块内部切换（如个人中心、服务咨询）优先使用嵌套路由 (`/*`)，避免使用复杂的内部状态控制显示。
- **路由守卫**: 涉及用户资产或隐私的页面（如 Profile, Repair）必须包裹在 `ProtectedRoute` 中。

### 5. 国际化 (i18n)
- **禁止硬编码**: 页面中所有可见文本必须通过 `t()` 函数从 JSON 语言包中读取。
- **结构化维护**: 翻译 key 必须遵循 `模块.子模块.属性` 的层级结构，严禁扁平化堆叠。
- **同步更新**: 修改代码中文本时，必须同时更新 `locales/zh.json` 和 `locales/en.json`。

### 6. 安全与防御
- **敏感数据**: 手机号、邮箱展示前必须调用 `maskPhone` 或 `maskEmail`。
- **错误边界**: 关键渲染路径（如 `App.tsx` 入口）必须包裹 `ErrorBoundary`。
- **输入校验**: 表单项必须通过 `rules` 进行格式校验，防止空提交或非法字符。

### 7. Git 与协作
- **提交规范**: 推荐使用 `feat:`, `fix:`, `docs:`, `style:`, `refactor:`, `perf:` 等前缀标识提交类型。
- **代码清理**: 提交前应检查并移除未使用的 import、变量以及调试用的 `console.log`。
